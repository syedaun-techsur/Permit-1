## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted query params (status, sortBy, search, cursor) and request bodies crossing into NestJS admin handlers |
| db→render | Stored user data (fullName, email, audit_log details) reflected back in API responses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of privilege | `AdminController` — all routes | mitigate | `@UseGuards(JwtAuthGuard, RolesGuard)` + `@Roles('admin')` on the class in `admin.controller.ts`; every request validates JWT and asserts role='admin' before any handler runs |
| T-05-02 | Tampering (SQL injection via sortBy) | `AdminService.getAllPermits` — TypeORM QueryBuilder `orderBy()` | mitigate | Whitelist of allowed sortBy column names in `admin.service.ts::getAllPermits` (array includes-check before calling `.orderBy()`); never interpolate raw `sortBy` param into query string |
| T-05-03 | Information disclosure (password_hash leak) | `AdminService.getUsers` / `createUser` / `updateUser` response | mitigate | TypeORM query explicitly selects named columns (`select(['user.id','user.fullName','user.email','user.role','user.isActive','user.createdAt','user.lastLoginAt'])`), never `SELECT *`, in `admin.service.ts` |
| T-05-04 | Tampering (CSV injection) | `AdminService.exportAuditLogCsv` — CSV cell values from audit_log details | mitigate | All cell values wrapped in double-quotes with internal double-quotes escaped (`"` → `""`) in `admin.service.ts::exportAuditLogCsv`; cells starting with `=`,`+`,`-`,`@` are prefixed with a tab to neutralize formula injection |
| T-05-05 | Repudiation | Admin actions (create/deactivate/role change) | mitigate | `audit_log` entry written inside the same DB transaction as the user mutation in `admin.service.ts::createUser`, `updateUser`; `actor_id` and `occurred_at` captured server-side, never trusted from client |
| T-05-06 | Denial of service (unbounded CSV export) | `GET /admin/audit-log/export` — streaming all rows | accept | Admin-only endpoint; rate-limiting at the API gateway / reverse proxy layer (owned by infra). Batch size of 500 rows per DB fetch limits memory pressure in `admin.service.ts::exportAuditLogCsv`. |
| T-05-07 | Elevation of privilege (SELF_DEACTIVATION bypass) | `AdminService.updateUser` | mitigate | Explicit check `if (dto.isActive === false && userId === actorId)` in `admin.service.ts::updateUser` throws `ConflictException({ code: 'SELF_DEACTIVATION' })` before any DB write |
</threat_model>

**MessagesService methods**:

`sendMessage(applicationId, senderId, dto)`:
- Load application; throw 403 if status === 'draft'
- Verify senderId is applicant (owns app) OR assigned reviewer OR admin; else 403
- Create Message entity with `{ applicationId, senderId, body: dto.body, sentAt: new Date() }`
- Save; reload with sender relation (for senderName + senderRole)
- Build and return MessageObject: `{ id, applicationId, senderId, senderName: sender.firstName + ' ' + sender.lastName, senderRole: sender.role, body, attachments: [], sentAt, readBy: [] }`
- Trigger notification to recipient (the other party): call `notificationsService.createNotification(recipientId, applicationId, \`New message on application #${ref} from ${senderName}.\`)`

`listMessages(applicationId, userId, cursor?: string, limit = 50)`:
- Verify user access to application
- Query `messages` table with `application_id = applicationId`, ordered by `sent_at ASC`, cursor pagination using `id` as cursor
- Join message_reads to compute readBy array
- Join message_attachments for each message
- Return `{ data: MessageObject[], nextCursor: string | null }`

`markRead(messageId, userId)`:
- Upsert into `message_reads`: `INSERT INTO message_reads (message_id, user_id) VALUES ($1, $2) ON CONFLICT DO NOTHING`
- Return 200

`getUnreadCount(applicationId, userId)`:
- Count messages in application not in `message_reads` for this userId
- Return `{ unreadCount: number }`

`getAttachmentUploadUrl(messageId, reviewerId, dto)`:
- Verify reviewer owns or is assigned to the application
- Allowed mimeTypes: `['application/pdf', 'image/jpeg', 'image/png', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document']`; max sizeBytes = 26214400 (25MB); max 5 attachments per message (check count)
- storageKey = `message-attachments/${message.applicationId}/${messageId}/${uuidv4()}.${ext}`
- Call `s3Service.getPresignedPutUrl(storageKey, dto.mimeType, 900)` (15-min expiry)
- Return `{ uploadUrl, storageKey, expiresAt }`

`registerAttachment(messageId, reviewerId, dto)`:
- Verify reviewer access
- Create and save MessageAttachment entity
- Return MessageAttachment

**MessagesController routes** (all under `@Controller('permits/:appId')`):

```
GET    /permits/:appId/messages                           → listMessages (A, R, ADM)
POST   /permits/:appId/messages                          → sendMessage (A, R)
GET    /permits/:appId/messages/unread-count             → getUnreadCount (A, R)
POST   /permits/:appId/messages/:msgId/read              → markRead (A, R)
```

**Part A — Extend NotificationsController and NotificationsService**:

The `notifications` table already exists from Phase 2. The `getUnreadCount()` method already exists. Add:

`NotificationsService.listNotifications(userId, cursor?, limit=50)`:
- Query notifications where `user_id = userId`, ordered by `created_at DESC`
- Cursor-based pagination using `id` as cursor
- Return `{ data: Notification[], nextCursor: string | null }`

`NotificationsService.markOneRead(notifId, userId)`:
- Update `notifications` set `is_read = true` where `id = notifId AND user_id = userId`
- Throw 404 if not found or not owned

`NotificationsService.markAllRead(userId)`:
- Update `notifications` set `is_read = true` where `user_id = userId AND is_read = false`
- Return `{ updated: number }`

`NotificationsController` new routes:

```
GET   /notifications                    → listNotifications (A, R)
PATCH /notifications/:notifId/read      → markOneRead (A, R)
PATCH /notifications/read-all           → markAllRead (A, R)
```

IMPORTANT: `PATCH /notifications/read-all` MUST be declared before `PATCH /notifications/:notifId/read` to prevent NestJS from treating "read-all" as a notifId param.

**Part B — Document archive endpoint (DOCS-05)**:

Install `archiver` npm package: `cd backend && npm install archiver && npm install --save-dev @types/archiver`

`DocumentsService.getArchiveUrl(applicationId, requesterId)`:
- Verify requester is assigned reviewer or admin for this application; throw 403 if not
- Load all active (non-deleted) documents for the application
- For each document, call `s3Service.getPresignedGetUrl(doc.storageKey, 3600)` to get a temp download URL
- Stream each document from MinIO via HTTP GET into an `archiver` ZIP stream
- Upload the resulting ZIP to MinIO at key `archives/${applicationId}/${uuidv4()}.zip`
- Generate a presigned GET URL for the ZIP (15-minute expiry)
- Return `{ downloadUrl: string, expiresAt: string }`

NOTE: If streaming ZIP through NestJS is complex, use a simpler alternative: generate the archive synchronously in memory using `archiver` with `finalize()` and write to a temp Buffer, then upload via `s3Service.uploadBuffer(key, buffer, 'application/zip')`. Add `uploadBuffer()` to S3Service if not already present.

`DocumentsController` new route:
```
GET /permits/:id/documents/archive   → getArchiveUrl  (R, ADM)
```

This route MUST be declared before `GET /permits/:id/documents/:docId/url` to avoid param conflict with "archive" being matched as a docId.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted message body (1-5000 chars) and attachment metadata entering MessagesController |
| client→API | Untrusted applicationId and msgId path params entering MessagesService — IDOR risk if ownership not checked |
| client→API | Untrusted notifId path param entering NotificationsController — IDOR risk if ownership not verified |
| db→render | Messages body stored in DB and returned to frontend — XSS risk if rendered as HTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Elevation of privilege | POST /permits/:appId/messages — non-party sending messages | mitigate | `MessagesService.sendMessage()` checks senderId is applicant (owns app) OR assigned reviewer OR admin; throws 403 ForbiddenException on mismatch |
| T-03-09 | Information disclosure | GET /permits/:appId/messages — third-party reading messages | mitigate | `MessagesService.listMessages()` verifies userId is applicant (owns app) OR assigned reviewer OR admin before executing query; returns 403 otherwise |
| T-03-10 | Information disclosure | IDOR on PATCH /notifications/:notifId/read — reading another user's notification | mitigate | `NotificationsService.markOneRead()` includes `AND user_id = userId` in WHERE clause; throws 404 if not found (prevents oracle-style enumeration) |
| T-03-11 | Elevation of privilege | POST attachment/upload-url — applicant getting reviewer-only upload URL | mitigate | `@Roles(UserRole.REVIEWER, UserRole.ADMIN)` decorator on `getAttachmentUploadUrl` controller method; RolesGuard enforces before service call |
| T-03-12 | Information disclosure | GET /permits/:id/documents/archive — applicant downloading reviewer-restricted ZIP | mitigate | `DocumentsService.getArchiveUrl()` verifies requester is assigned reviewer or admin; throws 403 if not; route also decorated with `@Roles(UserRole.REVIEWER, UserRole.ADMIN)` |
| T-03-13 | Tampering | Oversized message body (DoS via large payloads) | mitigate | `SendMessageDto.body` has `@MaxLength(5000)` decorator; NestJS global ValidationPipe rejects payloads exceeding limit with 400 |
| T-03-14 | Tampering | Unsafe MIME type in attachment upload — attacker registers .exe as application/pdf | mitigate | `MessagesService.getAttachmentUploadUrl()` validates mimeType against allowlist `['application/pdf','image/jpeg','image/png','application/vnd.openxmlformats-officedocument.wordprocessingml.document']` and throws 422 if not in list |
</threat_model>

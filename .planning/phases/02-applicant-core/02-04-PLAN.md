**`frontend/src/api/documents.api.ts`** — use the Axios instance from Phase 1:

```typescript
import { apiClient } from '../lib/axios';  // Phase 1 Axios instance with JWT interceptor
import type { PermitDocument, UploadUrlResponse } from '../types/document.types';

export const documentsApi = {
  getUploadUrl: (
    applicationId: string,
    payload: { filename: string; mimeType: string; sizeBytes: number },
  ) =>
    apiClient.post<UploadUrlResponse>(
      `/permits/${applicationId}/documents/upload-url`,
      payload,
    ),

  registerDocument: (
    applicationId: string,
    payload: {
      filename: string;
      mimeType: string;
      sizeBytes: number;
      documentType?: string;
      storageKey: string;
    },
  ) =>
    apiClient.post<PermitDocument>(`/permits/${applicationId}/documents`, payload),

  listDocuments: (applicationId: string) =>
    apiClient.get<PermitDocument[]>(`/permits/${applicationId}/documents`),

  getDocumentUrl: (applicationId: string, docId: string) =>
    apiClient.get<{ url: string; expiresAt: string }>(
      `/permits/${applicationId}/documents/${docId}/url`,
    ),

  deleteDocument: (applicationId: string, docId: string) =>
    apiClient.delete(`/permits/${applicationId}/documents/${docId}`),
};
```

**`frontend/src/hooks/useDocumentUpload.ts`**:

This hook manages the complete lifecycle for a batch of files being uploaded to a single application. It uses `axios` (not `fetch`) for progress events.

```typescript
interface UseDocumentUploadOptions {
  applicationId: string;
  onUploadComplete?: (doc: PermitDocument) => void;
  onUploadError?: (filename: string, error: string) => void;
}

interface UseDocumentUploadReturn {
  uploadFiles: (files: File[]) => Promise<void>;
  documents: PermitDocument[];            // Successfully uploaded docs
  uploadQueue: UploadFileState[];         // In-flight uploads
  fetchDocuments: () => Promise<void>;    // Refresh from API
  removeDocument: (docId: string) => Promise<void>;
  isLoadingDocuments: boolean;
}
```

Implementation:

1. **Client-side validation** (before any network request):
   - Allowed MIME types: `['application/pdf', 'image/jpeg', 'image/png', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document']`
   - Max file size: 26,214,400 bytes (25 MB)
   - Max filename length: 255 chars
   - If invalid: set file's UploadFileState to `{ status: 'error', error: 'File too large (max 25 MB)' }` or `{ status: 'error', error: 'Invalid file type' }` — do NOT make any network call for invalid files

2. **Upload flow** for each valid file (run in parallel with `Promise.allSettled`):
   ```typescript
   // a) Set status: 'uploading', progress: 0
   // b) POST /permits/:id/documents/upload-url → get { uploadUrl, storageKey }
   // c) PUT uploadUrl with file binary using axios (plain axios, NOT apiClient — no auth header on MinIO):
   //    axios.put(uploadUrl, file, {
   //      headers: { 'Content-Type': file.type },
   //      onUploadProgress: (e) => updateProgress(filename, Math.round(e.loaded / e.total * 100))
   //    })
   // d) POST /permits/:id/documents with { filename, mimeType, sizeBytes, storageKey } → get PermitDocument
   // e) Set status: 'uploaded', add to documents array
   // On any step failure: set status: 'error', error: error message — do not block other files
   ```

3. **removeDocument**: calls documentsApi.deleteDocument() then removes from documents array.

4. **fetchDocuments**: calls documentsApi.listDocuments() and sets the documents array.

Install `react-dropzone` and `react-pdf` if not already installed:
```bash
cd frontend && npm install react-dropzone react-pdf
```

Note: Use plain `axios` (not `apiClient`) for the presigned PUT to MinIO — the MinIO presigned URL includes authentication in query params; adding the JWT Authorization header from apiClient would break the request.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | filename, mimeType, sizeBytes from browser FileList crossing into upload-url and register endpoints |
| client→MinIO | Browser PUTs file directly to presigned MinIO URL; authentication is embedded in URL signature |
| db→render | Document presigned GET URL returned to browser — must verify applicant ownership before generating |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-19 | Tampering | Browser sends wrong sizeBytes or mimeType to get a presigned URL for a type/size that would be blocked | transfer | Server validates sizeBytes max (26,214,400) and mimeType `@IsIn([...])` in `UploadUrlRequestDto` (02-02); client-side check is UX-only; server is the authoritative guard |
| T-02-20 | Information disclosure | getDocumentUrl called for document belonging to another applicant | transfer | `DocumentsService.getDocumentUrl()` (02-02) verifies application.applicant_id === req.user.id before generating presigned GET URL |
| T-02-21 | Tampering | Client sends arbitrary storageKey in registerDocument to point to another tenant's object | accept | storageKey is opaque to the client in normal flow (generated server-side in getUploadUrl); a malicious client could supply a fabricated key to register a metadata entry pointing at wrong object; accepted in Phase 2 — full server-side MinIO event verification is Phase 3 hardening (HMAC-signed storageKey pattern) |
| T-02-22 | Denial of service | Attacker rapidly uploads many files to exhaust MinIO storage | transfer | Server enforces max 20 files and 100 MB total per application in `DocumentsService.getUploadUrl()` (02-02); throttling on submit endpoint limits rapid cycling |
| T-02-23 | Information disclosure | react-pdf renders PDF from presigned URL in browser — URL logged in network tab | accept | Presigned URL expires in 15 minutes; no persistent token leaked; browser network tab access requires physical device access; accepted |
</threat_model>

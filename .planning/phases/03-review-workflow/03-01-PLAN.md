**SQL migration file** `backend/src/database/migrations/003_phase3_messages.sql`:

Run this migration using the same pattern as 002_phase2_permits.sql (QueryRunner or raw SQL executed at startup). The migration MUST be idempotent — use `CREATE TABLE IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS`.

```sql
-- messages table
CREATE TABLE IF NOT EXISTS messages (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  application_id UUID NOT NULL REFERENCES permit_applications(id) ON DELETE CASCADE,
  sender_id      UUID NOT NULL REFERENCES users(id),
  body           TEXT NOT NULL,
  sent_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_messages_application_id ON messages(application_id);
CREATE INDEX IF NOT EXISTS idx_messages_sender_id ON messages(sender_id);
CREATE INDEX IF NOT EXISTS idx_messages_sent_at ON messages(sent_at);

-- message_reads table
CREATE TABLE IF NOT EXISTS message_reads (
  message_id UUID NOT NULL REFERENCES messages(id) ON DELETE CASCADE,
  user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  read_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  PRIMARY KEY (message_id, user_id)
);
CREATE INDEX IF NOT EXISTS idx_mr_user_id ON message_reads(user_id);

-- message_attachments table
CREATE TABLE IF NOT EXISTS message_attachments (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  message_id   UUID NOT NULL REFERENCES messages(id) ON DELETE CASCADE,
  filename     VARCHAR(255) NOT NULL,
  mime_type    VARCHAR(100) NOT NULL,
  size_bytes   BIGINT NOT NULL,
  storage_key  TEXT NOT NULL,
  uploaded_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_ma_message_id ON message_attachments(message_id);
```

**Step 3 — PermitsService: add four methods**:

`beginReview(id: string, reviewerId: string)`:
- Load permit; throw 409 `ConflictException` if status !== 'submitted' AND status !== 'additional_info_needed'
- Set `reviewer_id = reviewerId` if not already set; set `status = 'under_review'`
- Save permit; call `lifecycleService.createStage(id, 'under_review')`
- Call `auditService.createEntry(id, reviewerId, 'REVIEW_STARTED')`
- Call `notificationsService.createNotification(permit.applicantId, id, \`Your permit application #${permit.referenceNumber} is now under review.\`)`
- Return updated ApplicationObject

`requestInfo(id: string, reviewerId: string, dto: RequestInfoDto)`:
- Load permit; throw 409 if status !== 'under_review'
- Set `status = 'additional_info_needed'`, `info_request_note = dto.infoRequestNote`, `info_request_at = new Date()`
- Save; call `lifecycleService.createStage(id, 'additional_info_needed')`
- Call `auditService.createEntry(id, reviewerId, 'INFO_REQUESTED')`
- Excerpt the note (first 100 chars); call `notificationsService.createNotification(permit.applicantId, id, \`Additional information needed for #${permit.referenceNumber}: "${excerpt}"\`)`
- Return updated ApplicationObject

`respondToInfo(id: string, applicantId: string, dto: RespondToInfoDto, hasNewDocuments: boolean)`:
- Load permit; throw 409 if status !== 'additional_info_needed'
- Throw 422 `UnprocessableEntityException` if `!dto.responseNote && !hasNewDocuments`
- Set `status = 'under_review'`, `info_response_note = dto.responseNote ?? null`, `info_response_at = new Date()`
- Save; call `lifecycleService.createStage(id, 'under_review')`
- Call `auditService.createEntry(id, applicantId, 'INFO_PROVIDED')`
- If `permit.reviewerId`: `notificationsService.createNotification(permit.reviewerId, id, \`Applicant has responded to your information request on #${permit.referenceNumber}.\`)`
- Return updated ApplicationObject

`decide(id: string, reviewerId: string, dto: DecideDto)`:
- Load permit; throw 409 if status !== 'under_review'
- Throw 403 `ForbiddenException` if `permit.reviewerId !== reviewerId` (unless caller is admin — pass `isAdmin: boolean` param)
- Set `status = dto.outcome`, `decision_outcome = dto.outcome`, `decision_reason = dto.decisionReason`, `decision_at = new Date()`, `decided_by = reviewerId`
- Save; call `lifecycleService.createStage(id, dto.outcome)`
- Call `auditService.createEntry(id, reviewerId, dto.outcome === 'approved' ? 'APPLICATION_APPROVED' : 'APPLICATION_REJECTED')`
- Notify applicant: if approved → `"Your permit application #${ref} has been approved."`; if rejected → `"Your permit application #${ref} has been rejected: ${excerpt}"`
- Return updated ApplicationObject

**Step 4 — PermitsController: add action routes**:

All four routes are nested under `/permits/:id/actions/` using `@Post(':id/actions/begin-review')` etc. Apply existing `@UseGuards(JwtAuthGuard, RolesGuard)` pattern. Guard roles:

For `respond-to-info`, pass `hasNewDocuments: false` (documents check is out of scope for the basic controller call — the service will accept if responseNote is provided).

| T-03-03 | Elevation of privilege | POST /permits/:id/actions/decide — non-assigned reviewer deciding | mitigate | `PermitsService.decide()` checks `permit.reviewerId === reviewerId` before allowing; throws 403 ForbiddenException; admin bypass passed as explicit `isAdmin` flag from controller, not derived from input |
